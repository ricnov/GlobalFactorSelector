Global Factor Selector (Iterative Double-Selection LASSO)OverviewThe GlobalFactorSelector is a Python class designed to systematically evaluate and select explanatory factors from a large candidate pool (the "Factor Zoo"). It utilizes the Iterative Double-Selection LASSO methodology to evaluate new candidate factors against a baseline set of control factors.This implementation includes two critical improvements for statistical robustness:Feature Scaling: Standardizes inputs to ensure the LASSO $L1$ penalty is applied uniformly across features of varying magnitudes.Multiple Testing Correction: Implements Bonferroni correction to dynamically adjust $t$-statistic thresholds, preventing false positive discoveries as the number of tested candidates grows.Mathematical ContextThe Double-Selection LASSO aims to accurately estimate the significance of a candidate factor ($g$) on target returns ($r_{bar}$) while controlling for existing factors ($H$) and mitigating omitted variable bias.1. The Double-Selection LogicFor every candidate factor, the model performs two LASSO selections:Stage 1: Regress the target $r_{bar}$ on the control set $H$ to identify predictive controls ($I_1$).$$r_{bar} = \gamma' H + \epsilon$$Stage 2: Regress the candidate $g$ on the control set $H$ to identify controls correlated with the candidate ($I_2$).$$g = \delta' H + \eta$$2. Post-Selection OLSThe final unbiased evaluation regresses the target on the candidate factor and the union of the controls selected in Stage 1 and Stage 2 ($I_1 \cup I_2$).$$r_{bar} = \beta g + \theta' H_{I_1 \cup I_2} + u$$3. Bonferroni CorrectionTo control the Family-Wise Error Rate (FWER) when testing $m$ candidates, the adjusted significance level $\alpha_{adj}$ is calculated as:$$\alpha_{adj} = \frac{\alpha}{m}$$Class Reference: GlobalFactorSelectorInitialization ParametersParameterTypeDefaultDescriptionr_barnp.array-Target variable (e.g., risk premia).Hnp.array-Initial set of baseline control factors.Gnp.array-Pool of candidate factors to be evaluated.scale_inputsboolFalseActivates $Z$-score standardization for $H$ and $G$.adjustment_methodstr'none'Set to 'bonferroni' to dynamically penalize the $t$-threshold.significance_levelfloat0.10Base $\alpha$ used for the Bonferroni calculation.Core Methods_prepare_data(): Handles internal data scaling using StandardScaler. Note: $r_{bar}$ is never scaled to maintain the interpretability of risk premia._get_dynamic_threshold(): Calculates the critical $t$-value based on the remaining number of candidates.run(): The main execution engine.
